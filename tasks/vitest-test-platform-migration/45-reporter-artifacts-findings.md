# Reporter And Artifacts Findings

## Probe Summary

Temporary Vitest v4.1.6 probe files were created under this task directory and
removed after the spike.

The probe used:

- one passing test with a scenario record attachment
- one intentional failing test with a larger scenario record attachment
- built-in `agent` reporter
- `json` reporter
- `junit` reporter
- `annotate()` with JSON attachment files

## Findings

- Built-in `agent` reporter output is compact and suitable as the default human
  and agent terminal reporter.
- On failure, `agent` still prints the failing project, file, test name,
  assertion diff, and source line.
- `annotate()` is visible in terminal failure output as a notice.
- `annotate()` with an attachment copies the file into `attachmentsDir`.
- JUnit output preserves annotation text as `<property name="notice" ...>`.
- JSON output captured assertion failures, test names, and source stacks, but
  did not expose a useful attachment path in the observed output.
- Reporter and output file settings should be controlled at root/CLI level.
  The project-local reporter config in the probe did not produce JSON/JUnit
  files as expected.
- `outputFile` paths were resolved relative to the Vitest root in the probe.
  Final config should use root-relative paths carefully or absolute resolved
  paths to avoid nested `.result` locations.

## Design Consequences

- Use `agent` as the default terminal reporter.
- Use root-level or CLI-level `json`/`junit` reporter output for CI artifacts.
- Store detailed scenario records under each suite `.result` directory.
- On scenario failure, write a compact terminal context and include the artifact
  path in either the annotation message or the thrown error message.
- Keep full records out of the terminal by default.
- Use Vitest `attachmentsDir` if attachments are useful for HTML/blob-style
  workflows, but do not rely on JSON/JUnit consumers discovering attachment
  paths automatically.

## Recommended Scenario Failure Shape

```text
Scenario context:
  prId=303
  joinerUserId=404
  artifact=tests/scenario/.result/scenario-records/<name>.json
```

The full JSON record should contain every `ctx.record()` value. The terminal
should show a small configured subset or a truncation.

## Remaining Implementation Checks

- Confirm final root Vitest config can write per-project JSON/JUnit outputs
  under `apps/backend/.result/`, `apps/frontend/.result/`, and
  `tests/scenario/.result/`.
- Decide whether scenario records should be generated by helper code only or by
  a small custom reporter.
