# L1 Plan (High-Level Strategy)

## Goal Summary
- Keep the current homepage order intact.
- Make the Event Section read as the first real way to start using the product.
- Turn the current lower action section into a true fallback create path.
- Reduce footer noise and improve homepage resilience when event data is missing or failing.

## Staged Delivery

### Stage 1 — Structural Polish
- Do not reopen Hero semantics.
- Reframe the lower section around unmet demand and Community PR creation.
- Reduce footer links to three or fewer.
- Improve Event Section copy/fallback behavior so the first key path remains persuasive even when data is weak.
- Keep the implementation low-risk and aligned with the existing component architecture.

### Stage 2 — Event-First Conversion Polish
- Increase the sense that “these activities can be joined now”.
- Strengthen the bridge from Event Section into event detail / Anchor participation.
- Increase scene-binding and reduce any remaining “generic product module” feeling in the Event Section.

### Stage 3 — Micro-Delight & Retention
- Tune motion, pacing, and section handoff.
- Revisit bookmark/return nudges so they support, rather than interrupt, the landing narrative.
- Add or refine analytics to measure whether the page is actually helping users enter key paths.

## Implementation Principle
- Optimize the existing landing skeleton instead of rewriting the homepage from scratch.
- Prefer copy, hierarchy, and lighter styling changes before introducing new structural complexity.
- Keep each stage independently shippable.
