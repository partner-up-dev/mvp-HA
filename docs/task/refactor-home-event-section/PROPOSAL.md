# 首页 Event Section 重构提案（MVP 冷启动版）

## 1. 本次调整原则（基于最新共识）

- `Event Gateway` 与 `Event Conversion` 合并为同一入口层。
- event 区整体控制在 **2 个版面以内**。
- 每个活动 Campaign 在移动端按 **半版面（约 50vh）** 设计。
- 不使用模板化 `EventCard` 动态批量生成首页活动内容。
- 当前阶段以冷启动为前提，核心用户为大学生，不做“吸引力提升”验证闭环。

## 2. 用户与场景（第一性原理）

当前首页 event 区要解决的不是“信息完整”，而是“让大学生愿意点第一下”：

- 时间碎片化：课间、饭后、晚自习后快速浏览。
- 社交顾虑高：怕尴尬、怕门槛、怕一上来太正式。
- 决策偏快：看 1-2 屏就会决定继续点或离开。

因此策略应是：

- 2 屏内给足“我可以现在加入”的信号。
- 每个活动直接传达一个明确场景，不讲泛化口号。
- CTA 清晰且低成本，不让用户思考下一步该去哪。

## 3. 信息架构（最多 2 个版面）

### 版面 1：Gateway + Campaign A/B

结构从上到下：

1. 轻量 Gateway 条（与 Conversion 合并）
2. 羽毛球半版面
3. 慢跑半版面

Gateway 条样式要求：

- 一行 `body-large` 文案 + arrow icon。
- 文案示例：`去活动广场看全部活动`。
- 背景做一次性渐变闪烁（仅首次进入触发一次，约 700-900ms）。
- 点击进入 `/events`。

### 版面 2：Campaign C/D

结构从上到下：

1. 茶话会半版面
2. 口语练习半版面

补充：

- 在版面顶部可保留轻量锚点 pill（羽毛球/慢跑/茶话会/口语），用于快速跳转到对应半屏区域。
- 即使保留锚点，也不新增第 3 屏。

## 4. 四个 Campaign 方向（大学生版）

> 每个 Campaign 均为独立组件与独立内容，不走同模版循环渲染。

### 4.1 羽毛球（课后破冰局）

- 场景：晚课后、操场/体育馆、40-60 分钟出汗局。
- 关键词：轻对抗、快速熟络、零社交负担。
- 文案示例：
  - 标题：`下课就能开打`。
  - 副文案：`不用组熟人局，来了就能上场。`
  - CTA：`就在<地点轮播，打字机效果>`。

### 4.2 慢跑（晚风陪跑局）

- 场景：操场 3-5 圈，主打“有人一起更容易出门”。
- 关键词：不卷配速、轻陪伴、低压力。
- 文案示例：
  - 标题：`<地点轮播>，慢慢跑`。
  - 副文案：`不拼速度，先把第一圈跑起来。`
  - CTA：`就<最近一场的开始时间>`。

### 4.3 茶话会（宿舍外轻社交）

- 场景：奶茶店/校园角落，15-40 分钟短聊。
- 关键词：轻松聊天、认识同频同学、低门槛见面。
- 文案示例：
  - 标题：`喝杯茶，认识新同学`。
  - 副文案：`按兴趣匹配，奶茶我们包。`
  - CTA：`就在<地点轮播>`。

### 4.4 口语练习（开口训练局）

- 场景：四六级/雅思托福/面试前的开口练习。
- 关键词：敢开口、互相纠错、持续练习。
- 文案示例：
  - 标题：`先开口，再进步`。
  - 副文案：`不求完美发音，先把表达说出来。`
  - CTA：`看口语练习场次`。

## 5. 视觉与微动效策略（MVP 可落地）

- 每个半版面只保留：1 个主视觉 + 1 段短文案 + 1 个 CTA。
- 动效节奏：
  - Campaign 入场：`opacity + translateY`（180-260ms）。
  - CTA 反馈：轻微位移/亮度变化。
  - Gateway：一次性渐变闪烁，避免持续吸引造成噪音。
- 统一支持 `prefers-reduced-motion: reduce`，关闭非必要动画。

## 6. 内容与数据关系

采用双层策略：

- 营销层（首页）：4 个活动的固定 campaign 内容（静态策划）。
- 事实层（API）：从 `/api/events` 读取可加入状态与目标 eventId。

建议兜底：

- 若活动暂时无可用场次，CTA 仍可跳 `/events`，并展示 `场次更新中`。
- 冷启动阶段不追求复杂推荐排序，保持稳定可预期。

## 7. 组件改造建议

### 7.1 页面结构

- 替换当前 `HomeEventHighlights` + `HomeEventPlazaEntry` 组合。
- 新增 2 个 event slide 容器组件：
  - `HomeEventSlideOne.vue`（Gateway + 羽毛球 + 慢跑）
  - `HomeEventSlideTwo.vue`（茶话会 + 口语练习）

### 7.2 活动组件（独立）

- `HomeEventBadmintonHalf.vue`
- `HomeEventRunningHalf.vue`
- `HomeEventTeaTalkHalf.vue`
- `HomeEventSpeakingHalf.vue`

约束：

- 4 个组件可复用基础样式 token，但模板结构、文案、素材、动效均独立定义。
- 不允许 `v-for + 通用卡片模板` 直接生成四个 Campaign 主体。

### 7.3 i18n 与素材

- 在 `apps/frontend/src/locales/zh-CN.jsonc` 增加 `home.landing.eventCampaigns.*`。
- 首页素材建议放在：`apps/frontend/src/assets/home/events/`。
- 移动端资源控制：单图优先 WebP/AVIF，建议 120-220KB。

## 8. 实施阶段（MVP）

### Phase 1：结构落地

- 完成 2 个版面骨架。
- 接入 Gateway 合并入口。
- 完成 4 个半版面组件占位内容。

### Phase 2：视觉与动效

- 替换占位图为正式素材。
- 完成各 Campaign 微动效与可访问性降级。

### Phase 3：API 联动与验收

- 绑定 `/api/events` 的可用状态与跳转目标。
- 在移动端主流宽度（360/390/430）完成视觉验收。
- 通过构建：`pnpm --filter @partner-up-dev/frontend build`。

## 9. 验收标准（DoD）

- event 区总版面数 <= 2。
- 首页存在轻量 Gateway（body large + arrow + 一次性渐变闪烁）。
- 4 个活动均为独立半版面 Campaign，不使用 `EventCard` 模板化主展示。
- 4 个 Campaign 均具备面向大学生场景的差异化文案与视觉语义。
- 任一 Campaign 点击可进入活动详情或活动广场兜底路径。
- `prefers-reduced-motion` 下仍可完整浏览与点击。
