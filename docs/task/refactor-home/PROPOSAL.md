# 首页重构提案（Landing Page 化）

## 1. 背景

当前首页同时承载了品牌介绍、两条创建入口、自然语言表单、历史请求列表、联系作者入口。对移动端用户来说，信息密度偏高，进入页面后容易出现以下问题：

- 不知道先看什么，主线不清晰。
- 还没建立产品认知，就被表单和操作成本劝退。
- 小屏幕上“内容很多 + 层级不明确”的压迫感更明显。

目标用户里存在一类高流失人群：**“想找搭子，但暂时不知道找什么”**。他们需要先被启发和引导，而不是马上填写内容。

## 2. 目标与非目标

### 2.1 目标

- 将首页改造成明确的 Landing Page，先建立认知，再提供行动路径。
- 降低“不确定找什么搭子”的用户流失率，提升继续探索意愿。
- 在视觉与交互上保持克制，但通过大标题、留白、图像和轻动效提升吸引力。
- 让用户在浏览中段时更容易形成“后续再来”的动作（收藏网站/复制链接）。
- 新增并强化：
  - Event Highlights Section（活动亮点）
  - Event Plaza Entry（活动广场入口）

### 2.2 非目标

- 不在首页嵌入复杂表单（自然语言输入、结构化表单字段）。
- 不在本次重构中改动 PR 详情页、创建页核心业务逻辑。
- 不引入新的后端接口（优先复用现有 `useAnchorEvents`）。

## 3. 核心策略

### 3.1 首页角色调整

首页从“操作面板”调整为“品牌着陆页 + 导航中枢”，承担三件事：

- 让用户在 5 秒内明白这是什么（认知）。
- 给不确定需求的用户可点击、可探索的内容（启发）。
- 给准备行动的用户最短路径（转化）。

### 3.2 行为路径重排（从重到轻）

- 旧路径：进入首页 -> 看到复杂入口/表单 -> 犹豫 -> 离开
- 新路径：进入首页 -> 看懂价值 -> 浏览亮点 -> 进入活动广场/活动详情 -> 再决定创建或加入

## 4. 移动端 Landing 信息架构

建议首页按以下顺序组织（单列，首屏聚焦）：

1. Hero（大标题 + 一句价值陈述 + 主 CTA）
2. 快速认知区（3 个短句卡片，解释“这产品怎么用”）
3. Event Highlights Section（3~6 个精选活动卡片，优先可视化）
4. Event Plaza Entry（进入 `/events` 的主入口卡）
5. 收藏推荐区（滚动到中段后触发）
6. 次级动作区（发起搭子 / 联系作者）

说明：

- “我的搭子请求”从首页主视图下沉为次级入口（避免打断新用户认知）。
- 自然语言创建不在首页展开；保留能力但改为跳转/二级入口触达。

## 5. 区块方案（含文案方向）

### 5.1 Hero（首屏）

- 目标：快速建立产品认知与第一印象。
- 视觉：
  - 大字号艺术化标题（2 行以内）。
  - 1 张氛围图/插画（非复杂信息图）。
  - 强留白，避免信息堆叠。
- 建议文案方向（示例）：
  - 标题：`今天，不用一个人凑局`
  - 副标题：`把一句话变成可加入的搭子请求。`
  - 主 CTA：`看看正在发生的活动`
  - 次 CTA：`我想发起一个搭子`

### 5.2 快速认知区

- 3 个短句卡片，每卡 1 句，不超过 16 字：
  - `一句话发起`
  - `人数满即成团`
  - `分享后直接可加入`
- 以图标 + 短文案构成，不写长段落。

### 5.3 Event Highlights Section（新增重点）

- 数据来源：`useAnchorEvents`。
- 展示策略：
  - 移动端展示 3 张首屏外卡片（可纵向堆叠或横向滑动）。
  - 每卡只保留 3 类信息：封面、标题、1 行描述/标签。
  - 点击进入 `/events/:eventId`。
- 目标：为“不知道找什么搭子”的用户提供低认知成本的“现成选择”。

### 5.4 Event Plaza Entry（新增重点）

- 大卡片入口，独立于 Highlights，强调“查看更多”。
- 信息建议：
  - 标题：`去活动广场找灵感`
  - 描述：`按活动浏览，看看现在有什么可加入。`
  - 按钮：`进入活动广场`
- 路由：`/events`。

### 5.5 收藏推荐区（页面中段触发）

- 触发时机：
  - 用户滚动至页面约 45%~60%（推荐 50%）时触发。
  - 单用户每天最多提示 1 次（`localStorage` 节流）。
- 表现形式：
  - 轻量提示条/底部浮层（不全屏打断）。
  - 两个动作：
    - `收藏网站`（展示系统/微信环境操作指引）
    - `复制首页链接`（兜底）
- 环境文案区分：
  - 微信 WebView：`点击右上角 ··· 可收藏到微信。`
  - 普通浏览器：`将本站加入书签，随时回来找搭子。`

### 5.6 次级动作区

- 保留但弱化：
  - `发起搭子请求`（跳转 `/pr/new`）
  - `联系作者`（跳转 `/contact-author`）
- 放在页面底部，避免与探索路径竞争首屏注意力。

## 6. 交互与视觉原则（克制但引人入胜）

- 控制动作数量：每屏最多 1 个主按钮 + 1 个次按钮。
- 控制动效数量：仅保留入场淡入、卡片轻微上浮，禁用高频闪烁/打字机效果。
- 控制文字长度：一屏内正文总字数尽量不超过 70 字。
- 控制层级深度：减少折叠区和隐藏表单，避免“点开才发现更复杂”。
- 控制色彩：使用 1 个品牌主色 + 中性底色，突出关键操作，不做花哨渐变堆叠。

## 7. 实现计划（前端）

### 阶段 1：结构重排

- 重构 `HomePage.vue`：改为 Landing section 顺序布局。
- 将 `HomeNLCreatePanel` 从首页主流程移出（保留能力但不默认展开）。
- 新增 `HomeEventHighlights`、`HomeEventPlazaEntry`、`HomeBookmarkNudge` 组件。

### 阶段 2：数据与路由衔接

- `HomeEventHighlights` 复用 `useAnchorEvents`。
- 统一卡片点击路由到 `/events/:eventId`。
- Event Plaza Entry 跳转 `/events`。

### 阶段 3：埋点与验证

新增埋点（建议）：

- `home_hero_primary_click`
- `home_event_highlight_click`
- `home_event_plaza_entry_click`
- `home_bookmark_nudge_shown`
- `home_bookmark_action_click`

说明：需在 `shared/analytics/events.ts` 扩展事件白名单与 payload 类型。

### 阶段 4：文案与国际化

- 在 `locales/zh-CN.jsonc` 新增 landing 相关 key（hero、highlight、bookmark、plaza-entry）。
- 保持旧 key 兼容一段时间，避免一次性替换带来的风险。

## 8. 验收标准（面向“降低流失”）

- 首页首屏不出现复杂输入表单。
- 用户在 5 秒内可看到“产品是什么 + 下一步做什么”。
- 首页中段可触发收藏推荐，且有可执行动作（收藏引导或复制链接）。
- 首页存在 Event Highlights Section 与 Event Plaza Entry，且均可点击跳转。
- 移动端（主流宽度 360~430）无信息拥挤与明显视觉跳动。

## 9. 关键指标（上线后 1~2 周观察）

- 首页跳出率（或首屏无互动离开率）下降。
- 首页到 `/events` 点击率提升。
- 首页到 `/events/:eventId` 点击率提升。
- 首页收藏提示触发后的动作点击率（收藏/复制）达到可观水平。
- 新用户会话时长与页面滚动深度提升。

## 10. 风险与缓解

- 风险：老用户找不到“我的搭子请求”。
  - 缓解：在底部保留“我的搭子请求”入口（非默认大区块）。
- 风险：首页转化到创建页短期下降。
  - 缓解：保留 Hero 次 CTA“我想发起一个搭子”，并跟踪转化漏斗。
- 风险：收藏操作在不同浏览器能力不一致。
  - 缓解：统一提供“复制链接”作为兜底动作。
