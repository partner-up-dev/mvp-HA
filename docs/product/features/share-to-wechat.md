# 功能：微信内分享

## 用户故事

- 作为发起者，我想在微信内通过菜单分享 Community PR / Anchor PR，让对方直接看到分享卡片。
- 作为用户，我希望在首页、创建页、联系客服页也能通过微信菜单分享当前页面，避免只能分享 PR 详情页。

## 流程

- 用户在微信 WebView 打开首页、`/cpr/new`、`/contact-support`、`/cpr/:id` 或 `/apr/:id`。
- 前端请求后端生成 JS-SDK 签名并完成配置。
- 前端调用微信 JS-SDK，同时配置“发送给朋友”和“分享到朋友圈”两种分享数据。
- 首页、创建页、联系客服页使用页面级描述性文案（而非 URL）作为分享说明，并统一使用品牌 logo 作为缩略图。
- 分享落地链接会附加 `spm=<route>.wechat_share`，用于链路归因。
- 用户通过微信菜单分享，生成包含标题/缩略图的分享卡片。
- 被分享者点击卡片后进入对应页面（首页、Community PR 创建页、联系页、Community PR 页或 Anchor PR 页）。

## 验收标准

- 微信内可正常拉起分享菜单。
- 首页、创建页、联系客服页的分享卡片说明文本为页面描述文案，不显示为页面 URL。
- 首页、创建页、联系客服页的分享卡片缩略图为产品 logo。
- 分享卡片包含正确标题、摘要与缩略图。
- “发送给朋友”和“分享到朋友圈”均可使用。
- 分享链接可打开当前被分享页面（不仅限于 PR 页面；PR 页面分为 `/cpr/:id` 与 `/apr/:id`）。
