# 功能：客服与作者联系分流

## 用户故事

- 作为用户，我希望在首页次级动作区和各页面底部都能快速找到“联系作者”入口，便于反馈功能建议与缺陷。
- 作为用户，我希望在 `/contact-support` 能明确区分“联系客服”（报销/投诉）和“联系作者”（功能/缺陷反馈）。
- 作为用户，我希望在 `/contact-support` 还能看到“内测群”入口，并可在弹窗中扫码入群。
- 作为运营，我希望客服链接可由后端配置动态覆盖，并在配置加载中、失败或缺失时自动回退到默认链接，避免页面不可用。

## 流程

- 用户在首页底部次级动作区点击“联系作者”进入 `/contact-support`。
- 用户在任意页面底部点击“联系作者（功能建议 / Bug 反馈）”也进入 `/contact-support`。
- `/contact-support` 页面展示分流说明：
  - 报销、投诉用户 -> 联系客服
  - 希望加入内测群用户 -> 打开二维码弹窗扫码入群
  - 功能建议、缺陷反馈 -> 跳转 `/contact-author`
- `/contact-author` 为独立页面，展示作者微信二维码用于反馈。
- 联系页会检测当前环境：
  - 微信内：优先使用配置 `wecom_support_link_wechat_in`，缺失或非法时回退默认微信内链接。
  - 微信外：优先使用配置 `wecom_support_link_wechat_out`，缺失或非法时回退默认微信外链接。
- 用户点击“联系客服”按钮后，在新窗口打开目标客服链接。
- 用户点击“查看群二维码”后，在页面中央弹窗显示内测群二维码（配置缺失时显示空态提示）。

## 验收标准

- 首页和全局页脚均提供“联系作者”入口，且跳转到 `/contact-support`。
- `/contact-support` 页面存在“联系客服”主按钮和“联系作者”次级入口。
- `/contact-support` 页面存在“内测群”卡片入口，且视觉优先级低于客服/报销入口。
- `/contact-author` 为可访问页面，展示作者二维码（无配置时显示空态提示）。
- 内测群二维码配置 key 为 `wechat_beta_group_qr_code`，前端优先使用配置值并校验为合法 http/https URL。
- 联系页在后端配置缺失、为空或非法 URL 时仍可使用默认客服链接。
- 联系页在后端配置加载中或请求失败时仍可使用默认客服链接。
- 联系页在后端配置为合法 URL 时优先使用配置值。
- 微信内外环境分别命中对应客服链接，且页面无阻断报错。
