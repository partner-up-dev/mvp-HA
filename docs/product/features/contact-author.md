# 功能：联系客服（功能建议 / Bug 反馈）

## 用户故事

- 作为用户，我希望在首页次级动作区和各页面底部都能快速找到“联系客服”入口，便于反馈问题。
- 作为用户，我希望点击联系客服后可直接进入企业微信客服，无需再手动判断微信内外场景。
- 作为运营，我希望客服链接可由后端配置动态覆盖，并在配置缺失时自动回退到默认链接，避免页面不可用。

## 流程

- 用户在首页底部次级动作区点击“联系客服”进入 `/contact-support`。
- 用户在任意页面底部点击“联系客服（功能建议 / Bug 反馈）”也进入 `/contact-support`。
- 若用户访问旧路径 `/contact-author`，前端会自动重定向到 `/contact-support`，并保留 query/hash。
- 联系页会检测当前环境：
  - 微信内：优先使用配置 `wecom_support_link_wechat_in`，缺失或非法时回退默认微信内链接。
  - 微信外：优先使用配置 `wecom_support_link_wechat_out`，缺失或非法时回退默认微信外链接。
- 用户点击“联系客服”按钮后，在当前窗口直接打开目标客服链接。

## 验收标准

- 首页和全局页脚均提供“联系客服”入口，且跳转到 `/contact-support`。
- `/contact-author` 可兼容跳转到 `/contact-support`，且 query/hash 不丢失。
- 联系页在后端配置缺失、为空或非法 URL 时仍可使用默认客服链接。
- 联系页在后端配置为合法 URL 时优先使用配置值。
- 微信内外环境分别命中对应客服链接，且页面无阻断报错。
