# 功能：联系作者（功能建议 / Bug 反馈）

## 用户故事

- 作为用户，我希望在首页和各页面底部都能快速找到“联系作者”入口，便于反馈问题。
- 作为用户，我希望进入联系页后直接看到作者微信二维码，减少沟通成本。
- 作为作者，我希望二维码可通过后端配置动态更新，无需重新发版前端。

## 流程

- 用户在首页看到“功能建议 / 问题反馈”区块，点击“联系作者”进入 `/contact-author`。
- 用户在任意页面底部点击“联系作者（功能建议 / Bug 反馈）”也可进入 `/contact-author`。
- 联系页通过后端接口读取 `config` 表中的 `author_wechat_qr_code` 并展示二维码。
- 若后端未配置该 key，则联系页展示“暂未配置作者微信二维码”的提示文案。

## 验收标准

- 首页存在“功能建议 / 问题反馈”入口区块，且可跳转到 `/contact-author`。
- PR 详情页、PR 创建页、联系页底部均展示统一“联系作者”入口。
- `/contact-author` 页面二维码区域居中展示。
- 二维码来源于后端公共配置接口，读取 key `author_wechat_qr_code`。
- 当二维码配置为空或缺失时，页面展示空态提示而非报错崩溃。
