# 功能：企业微信自建应用消息创建搭子请求

## 用户故事

- 作为企业微信用户，我希望在自建应用里发一句话就能生成搭子请求草稿，并收到可继续发布的链接。

## 流程

- 企业微信自建应用回调消息到后端 `/api/wecom/message`。
- 后端解析消息内容并创建 `DRAFT` 搭子请求。
- 后端通过企业微信异步回复消息，返回草稿链接，并提示“打开链接发布后生成用户 PIN”。

## 验收标准

- 支持 URL 验证与加密消息回调。
- 文本消息可成功创建 `DRAFT` 搭子请求。
- 异步回复包含草稿链接与发布提示，不直接返回 PIN。

## Known Limitations

- 企业微信消息回调仅包含 UTC timestamp，不包含发送者时区。
- 为了给自然语言解析提供相对时间参考，后端当前按 `Asia/Shanghai` 推断 `nowWeekday`。
- 非 `Asia/Shanghai` 时区用户在“今天/周末”等相对时间语义上可能出现偏差。
