# 跨领域交互

## 1. PartnerRequest Core x Identity And Session

- Community PR 的发布、加入、退出依赖本地账户 + PIN 体系。
- Anchor PR 的关键动作依赖 authenticated 会话与微信绑定关系。
- 身份不是独立终点，而是协作链路的支撑条件。

## 2. PartnerRequest Core x Participation And Reliability

- PR 是否可加入、是否成团、是否满员、是否需要确认，都由参与与可靠性规则影响。
- 参与槽位状态会反过来影响 PR 当前人数、可用性与后续动作。

## 3. Event And Anchor Context x PartnerRequest Core

- Anchor Event 与 batch 为 Anchor PR 提供上下文和进入路径。
- Anchor PR 不是孤立对象，而是活动上下文中的一个协作实例。

## 4. Distribution And Attribution x PartnerRequest Core

- 分享能力将 PR 变成可传播对象。
- `spm` 归因把传播来源带回后续行为链路。
- 再进入是分享闭环的重要组成部分。

## 5. Participation And Reliability x Distribution And Attribution

- 分享能带来加入，但真正让协作可持续的是可靠性闭环。
- 通知、确认、签到把“传播成功”进一步转化为“协作真实发生”。

## 6. Support And Operations x All Other Domains

- 客服、作者反馈、内测群、配置项、管理端能力都会影响用户最终是否能完成链路。
- 这部分虽然不是所有用户的主路径，但对产品冷启动和服务可持续性很关键。
