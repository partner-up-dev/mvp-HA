# Frontend Gap Expansion (Snapshot: 2026-02-25)

## 1. 当前前端已具备能力
- 页面主流程：`/`、`/pr/:id`、`/pr/new`、`/contact-author`。
- 创建流程：自然语言创建 + 结构化创建（含 DRAFT/OPEN）。
- PR 详情：支持加入/退出、编辑内容、手动改状态、分享（链接/微信/小红书）。
- 微信自动登录基础：`useWeChatAutoLogin` + `/api/wechat/oauth/*` 会话探测与跳转。

## 2. 前端核心 GAP（按优先级）

### P0
- 状态语义不符合 PRD：
  - 仍以 `ACTIVE` 展示和操作，缺 `READY/FULL` 产品语义。
  - 未体现“达到 min 后仍可加入，仅显示 READY 徽标”的交互。
- Anchor vs Community 无前端分层：
  - 页面与操作上没有“Anchor PR”识别与专属规则。
  - “Anchor 参与需登录”没有在 join 前做条件门禁。
- 登录策略与 PRD 不一致：
  - 当前 `App.vue` 全局启用自动登录（微信内 + 配置开启时），是“全局强尝试”。
  - PRD 要求是“整体可选，仅 Anchor 参与强制”。
- 确认机制缺失：
  - join 后无确认按钮/状态。
  - 无 T-24h/T-2h 提醒后的前端承接态。
  - 无 T-1h 未确认释放、T-30min 锁定的倒计时/禁用提示交互。
- Slot 粒度交互缺失：
  - 仅展示聚合人数，无个人 `JOINED/CONFIRMED/RELEASED/ATTENDED` 可视化。

### P1
- Check-in 体验缺失：
  - 无“已到场”按钮。
  - 无“是否参加/是否愿再来”事后回传入口。
- Batching 承接缺失：
  - 当前批次满员时，无“附近同类场次”推荐区。
  - 无“新批次已开放”提示或跳转路径。
- 经济层展示缺失：
  - 无“平台已订场/已谈优惠/支付规则”模块。
  - 无 Model A/C 的用户可见解释模板。
- 报销入口缺失：
  - 无 PR=CLOSED 时“申请报销”按钮。
  - 无“复制 PR 链接并跳转企业微信”的前端引导流程。
- 安全入口缺失：
  - 无举报按钮与最小表单。
  - 无“仅公共场地”显式提示区块。
- 指标埋点缺失：
  - 未见 PV、join、min 达成、confirm、check-in、14 天复参与埋点事件调用。

## 3. 前端建议落点（实施顺序）
1. 先改状态表达层：UI 显示 `READY/FULL`（可由后端字段或前端派生），去除 `ACTIVE` 面向用户暴露。
2. 将登录从“全局自动”改为“Anchor Join 前置门禁 + 其他场景可跳过”。
3. 增加“加入 -> 确认 -> 锁定/释放”组件与倒计时提示。
4. 增加 Check-in 与事后反馈模块。
5. 增加支付/补贴/报销与安全模块。
6. 补齐关键行为埋点。
